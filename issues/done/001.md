## What to build

Create the uv-managed Python app scaffold for `network-scripts`, including the Typer CLI entry point and pytest setup. This should establish the supported command shape without migrating Cisco Device behavior yet.

## Acceptance criteria

- [x] `pyproject.toml` defines a package using the `src/` layout and a `network-scripts` CLI entry point.
- [x] `uv run network-scripts --help` works and shows top-level help.
- [x] `uv run network-scripts serial --help` and `uv run network-scripts cisco --help` work, even if Cisco commands are not implemented yet.
- [x] `uv run pytest` runs successfully.
- [x] Developer-facing docs/examples use `uv` commands, not `python` or `pip`.

## Blocked by

None - can start immediately
